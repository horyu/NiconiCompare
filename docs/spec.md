# NiconiCompare 機能仕様

NiconiCompare は、ニコニコ動画の視聴中に動画同士を比較し、好みをカテゴリ別のレーティングとランキングとして記録するブラウザ拡張機能です。

## 対象範囲

- Chrome / Firefox の Manifest V3 拡張機能として動作する。
- ニコニコ動画の watch ページを対象とする。
- データはブラウザのローカルストレージに保存し、外部サーバーへ送信しない。
- クラウド同期、他サイト対応、ストア公開の自動化は対象外とする。

## 比較とレーティング

- 視聴中の動画と、直近に扱った動画から選んだ比較対象を「良い・同じ・悪い」で評価する。
- 評価は `CompareEvent` として、現在動画、比較対象、判定、カテゴリ、日時を記録する。
- 同じ比較操作を続けた場合は直前イベントを更新し、比較対象やカテゴリを変更した後は新しいイベントを作成する。
- イベントを無効化・復活・判定変更・削除できる。削除は無効化済みイベントに対してのみ行う。
- レーティングはカテゴリ別に Glicko-2 で算出する。イベント追加時に更新し、必要に応じて全イベントから再計算できる。

## 動画情報と比較候補

- 動画タイトル、サムネイル、投稿者情報は watch ページの JSON-LD から取得する。
- 取得した動画・投稿者情報は再訪時に更新する。
- JSON-LD を取得できない場合、保存済みの動画情報があれば比較を継続する。保存済み情報もなければ比較入力を無効化する。
- 比較候補は直近に扱った動画を LRU として保持する。
- 比較対象はピン留めでき、ページ再読み込み後も維持する。
- 現在動画と比較対象が同じ場合は評価できない。

## カテゴリ

- カテゴリは比較の観点を表し、1イベントには1カテゴリを設定する。
- アクティブカテゴリは、以後に登録するイベントのカテゴリとして使用する。
- レーティングと一覧表示はカテゴリ単位で扱う。
- カテゴリの追加、名称変更、並び替え、削除、オーバーレイへの表示対象設定ができる。
- カテゴリ名は 1〜50 文字とし、Unicode の文字・数字、半角スペース、`・`、`_`、`-` を使用できる。

## UI

### オーバーレイ

- watch ページ上で現在動画と比較対象の情報、カテゴリ選択、評価ボタンを表示する。
- 設定により、オーバーレイと動画情報取得の有効・無効、自動クローズ時間、閉じた状態での直近判定表示を変更できる。
- カテゴリまたは比較対象を変更すると、表示中の直近判定をクリアする。
- オプションページを開く導線を提供する。

### Popup

- アクティブカテゴリの直近イベントを表示する。
- 動画ごとの勝・分・敗数の表示を切り替えられる。
- オーバーレイと動画情報取得の有効・無効を切り替えられる。
- オプションページを開く導線を提供する。

### Options

Options は次のタブで構成する。

- **動画一覧**: カテゴリ別レーティング、評価数、勝敗数、最終判定日時の表示、検索、並び替え、CSV / TSV 出力。
- **評価一覧**: イベントの検索、絞り込み、無効化・復活・判定変更・削除、CSV / TSV 出力。
- **カテゴリ**: カテゴリの追加、名称変更、並び替え、削除、オーバーレイ表示対象の設定。
- **設定**: 比較候補数、オーバーレイ、自動クローズ、一覧表示、Glicko-2 初期値の設定とレーティング再計算。
- **データ操作**: JSON のエクスポート・インポート、共有用 HTML 出力、孤立データの削除、全データ削除。

一覧のフィルタとページ位置は、同一セッション内でタブへ戻ったときに復元する。

## データ操作

- JSON エクスポートは `nc_*` データを一括で出力する。
- JSON インポート後は、インポートされたイベントからレーティングを再計算する。
- 現在のスキーマバージョンと互換性がないデータはインポートを拒否する。
- 共有用 HTML は、選択した単一カテゴリの有効イベントと動画一覧を、オフラインで閲覧できる単一ファイルとして出力する。
- 自動および手動クリーンアップでは、イベント・現在動画・比較候補・ピン留め対象から参照されない動画、投稿者、レーティングを削除する。

## 表示形式

- 画面上の日時は `YYYY/MM/DD HH:mm:ss` 形式で表示する。
- 出力ファイル名の日時は `YYYYMMDDHHmmss` 形式を使用する。
- 勝敗は現在動画の視点で `>`、`=`、`<` として表示する。

技術構成とデータ設計は [アーキテクチャ](architecture.md) を参照してください。
