# ドキュメントメンテナンスガイド

**Last Updated**: 2025-12-18

本ガイドは NiconiCompare のドキュメント群の関係性と更新ルールを定義する。

---

## 📖 ドキュメント構成

```
docs/
├── spec.md              # 「何を作るか」（機能要件・UI仕様。UI はオーバーレイ等の不変要件のみを記載し、細部は実装を参照）
├── architecture.md      # 「どう設計するか」（技術アーキテクチャ。Content Overlay など実装依存部分はコードへのリンクで補う）
├── ui-overlay.md        # オーバーレイ UI の実装スナップショット（DOM/スクショ参照用）
└── developer-guide.md   # 「どう作るか」（環境構築・手順）
```

各ドキュメントの詳細は[トップレベル README](../README.md#-ドキュメント)を参照。

---

## 📝 ドキュメント更新ルール

### 更新が必要な場合

| 変更内容             | 更新対象文書                               |
| -------------------- | ------------------------------------------ |
| 新機能の追加         | spec.md, architecture.md                   |
| 技術スタックの変更   | developer-guide.md, architecture.md        |
| バグ修正             | 通常は不要（重大な設計変更を伴う場合のみ） |
| UI/UX 変更           | spec.md                                    |
| パフォーマンス最適化 | architecture.md, developer-guide.md        |
| ビルド手順の変更     | developer-guide.md                         |
| 開発ツール・lint設定の変更 | developer-guide.md                   |

### 更新時の注意事項

1. **バージョン番号の更新**: 各文書の冒頭にある `Version` と `Last Updated` を更新
2. **改訂履歴の追記**: 文書末尾の「改訂履歴」セクションに変更内容を記録
3. **相互参照の確認**: 複数文書にまたがる変更の場合、すべての文書で整合性を確認
