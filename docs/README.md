# ドキュメント運用

各文書は次の責務に限定します。

| 文書                      | 内容                                       |
| ------------------------- | ------------------------------------------ |
| `README.md`               | プロジェクト概要とドキュメントへの導線     |
| `docs/spec.md`            | ユーザーに見える挙動とドメイン上の不変条件 |
| `docs/architecture.md`    | 実装済みの構成、データフロー、永続化設計   |
| `docs/developer-guide.md` | 環境構築、開発コマンド、開発規約           |

## 更新方針

- 同じ説明を複数文書に書かず、詳細を持つ文書へリンクする。
- 実装の変更と同じコミットで対応する文書を更新する。
- 実装予定、未計測の性能値、実装と異なる疑似コードは記載しない。
- 型・既定値・コマンドなどコードで確認できる詳細は、必要な範囲だけ記載する。
- 実装が正となる細かな UI 表現や文言は固定しない。
